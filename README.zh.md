## 项目背景

如今，我们开启一段事业的方式大致有两种：
要么**成立一家公司**，要么**加入一家公司**。

公司通过薪资体系，将人们组织起来，共同完成那些个人难以独立完成的伟大事业，同时为成员提供了相对稳定的收入。
然而，这种看似成熟的合作方式，在带来效率与秩序的同时，也存在以下几个长期困扰我们的问题：

1. **合作缺乏自由**

   我们常常在公司里被安排好一切——做什么、和谁做。
   想换一个团队、追求新的方向，却要经历繁琐的面试、放弃奖金，还得冒着下一份工作可能更糟的风险。

2. **工作中充斥着无意义的内容**

   许多工作任务的存在，并非源于真正的产品需求或个人信念，而是出于管理层的决策、流程或考核要求。
   其根本原因在于：**我们创造的产品并不属于我们自己**。因此，我们往往并不是在为产品本身负责，而是在为“工作”负责，为“流程”和“考核”负责。
   这使得那些渴望从工作中找到意义的人，尤为痛苦。

3. **收入与时间绑定**

   我们的收入来源于出售时间。一旦停止工作，收入也随之停止。因为产品的长期价值并不属于我们，离开公司后，我们与产品的未来收益再无关联。

4. **公司自身的高成本与管理困境**

   对公司而言，招聘合适的人极其困难且昂贵，需要耗费大量时间筛选与面试。
   员工入职后，由于上述问题，公司还需投入更多精力去“管理”与“激励”员工，而这些方式往往并不理想。

## 介绍

Copany 旨在为以上问题提供一种新的解决思路。
这或许不是完美的答案，但我们希望通过共同探索，逐步构建出一种比传统公司更自由、更高效、更公平的协作方式。

在 Copany 中：

1.  **人人都能参与或发起项目**
    无需面试、无门槛进入，随时加入、随时退出，合作完全自由。

2.  **项目收益按贡献积分分配**
    每个项目的收益将根据参与者的贡献积分占比来分配，积分永久有效。
    这意味着，你参与的项目，有一部分始终属于你。即使你离开项目，也依然能分享其未来收益。
    因为“成果属于自己”，人们自然会更专注于那些对项目真正重要的工作。

3.  **贡献积分以事项复杂度为依据**
    每完成一项符合标准的事项，就能获得与其复杂度对应的贡献积分。
    超出预期质量的成果，还可以获得额外奖励。

4.  **未来将支持更多形式的贡献**
    除了完成事项，Copany 还将支持其他贡献方式，例如资金投入或物资支持，并将这些形式转化为相应的贡献积分。

#### 为什么以「事项复杂度」衡量贡献？

让我们思考一个问题：
**“你为一个项目贡献的核心是什么？”**
答案是——时间与经验/知识。

在传统公司中，薪资体系的本质是“工作时间 × 经验匹配的时薪”。
但在 Copany 中，我们既不限制你的工作时间，也不通过面试来评估经验，因此无法采用这种方式衡量贡献。

于是，我们选择以「成果」作为衡量标准。
完成的事项，正是你投入时间与经验后产出的结果。
而一个事项所需的时间与其复杂度高度相关，因此通过“事项复杂度”即可相对公平地反映贡献。

在 Copany 中，只有达到既定质量标准的事项才会被视为有效交付并计入贡献积分。
由于完成事项所需的时间与其复杂度高度相关，且能否达标取决于执行者的经验与能力，因此经验因素已在体系中被间接体现。
这使得 Copany 能够通过“事项复杂度”这一单一维度，较为公平地衡量不同贡献者的投入。

对于超出标准的优秀交付，Copany 计划在未来引入额外奖励机制进行补充。

#### 复杂度的评判机制

在实践中，Copany 当前采用如下机制：

1. **事项负责人** 在任务开始前对事项复杂度进行初步评估；

2. **团队成员** 可在执行过程中讨论并随时修正；

3. **项目总负责人** 在项目完成后对事项结果与分级进行最终审核；

4. **所有记录公开透明**，任何人都可审阅与讨论，从而逐步形成更准确的复杂度共识。

通过不断实践、反馈与迭代，Copany 希望建立一套能真实反映个人价值与项目贡献的系统。

## 使命、愿景

**使命：**

创造出一种自由、透明、成果共享的协作方式。

**愿景：**

让人们的工作更加自由，且充满意义。

## 产品框架

**探索页（Explore）**

浏览并探索所有 Copany 项目，了解项目进展与参与方式。

**个人页（Profile）**

进行账户管理，包括收益接收方式的设置与结算记录查看。

**项目页（Project）**

一个项目包含以下四个核心区域：

- **讨论区（Discussion）**：

  分享想法、讨论方向、交流灵感。

- **合作区（Collaboration）**：

  规划项目事项，分配负责人，确定事项分级与执行顺序。

- **贡献记录区（Contribution）**：

  记录所有已完成的事项及其对应的贡献积分比例。

- **收益支出记录区（Finance）**：

  展示项目的收益与支出情况，并按照贡献比例定期分配收益。

## 如何贡献

**一、创建 Issue 并完成任务**

1. **开启讨论：**
   建议在创建 Issue 或开始工作前，先发起一次讨论，与其他成员充分交流想法。
2. **创建 Issue：**
   讨论达成共识后，创建一个或多个 Issue，指定负责人并设置复杂度分级。
3. **提交审核：**
   任务完成后，将 Issue 状态切换为 In Review，由 Copany 负责人审核完成内容及分级是否合理。
4. **确认完成：**
   审核通过后，将状态切换为 Done。成果（代码或设计等）合并至主分支，Issue 负责人将获得对应分级的贡献积分。

**二、申请现有 Issue 并完成**

1. 在合作区浏览当前已规划的 Issue，查看其内容、优先级与复杂度分级。

2. 在 Issue 的 Assignee 选项中，申请将自己分配为负责人。

3. 待 Copany 负责人、Issue 负责人或创建者审核通过后，即可开始执行任务。

4. 任务完成与审核流程同「创建 Issue 并完成」步骤一致。

## 收益分配机制

1. **设置收益接收方式：**
   前往「个人页」，配置收益接收渠道，目前支持：

   - 支付宝（Alipay）
   - Wise
     上传对应的收款码即可完成设置。

2. **生成收益分配记录：**
   每月 10 日 0 点（UTC 时间），系统自动统计：

   - 上一周期（上月 10 日 0 点至本月 10 日 0 点）项目的总收益与支出；
   - 所有贡献者在当月 1 日 0 点时的贡献比例；
     并据此生成收益分配记录。

3. **项目负责人进行收益分配：**
   项目负责人依据分配记录与各贡献者设置的收款方式进行收益发放，并上传分配凭证。

4. **贡献者确认收益：**
   分配凭证上传后，贡献者将收到通知，前往确认是否已收到收益。
   确认完成后，本次收益分配正式结束。
